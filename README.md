# Figma Code Connect Skill

A Claude skill that walks non-technical users (designers, PMs, design system owners) through setting up [Figma Code Connect](https://www.figma.com/developers/code-connect) from scratch, one step at a time.

---

## What it does

When a developer inspects a component in Figma's Dev Mode, Figma normally shows auto-generated code. Code Connect replaces that with the real code snippet from your team's actual codebase: the correct import, the correct props, populated with the values from whichever variant is selected.

This skill makes that setup accessible to anyone on your team, not just developers.

---

## What Code Connect requires

- A **Figma Organisation or Enterprise plan** (Dev Mode is not available on free or Professional plans)
- A codebase using one of the supported frameworks: React, Vue, SwiftUI, Jetpack Compose, Angular, or plain HTML
- Node.js installed on the user's machine

---

## What the skill covers

The skill walks users through the complete setup end-to-end:

1. **Preflight checks** - Claude Code installation, Figma plan, single vs. multi-platform setup
2. **Figma file key** - extracting it from the URL automatically
3. **Access token** - generating one in Figma Settings with the right scopes
4. **Config files** - generating `figma.config.json`, `tsconfig.json`, and `.env` with real values filled in (no placeholders)
5. **CLI installation** - `npm install @figma/code-connect` and adding publish scripts to `package.json`
6. **First component** - connecting a component by pasting its Figma URL and code name
7. **Publishing** - running `npm run publish` and confirming the snippet appears in Dev Mode
8. **Refining** - adding the correct import path and mapping Figma properties to code props
9. **Scaling up** - connecting more components one at a time or in bulk

It also handles common errors inline: invalid URLs, missing parsers, auth failures, and snippets that don't appear after publishing.

## Multi-platform support

A single Figma component can be connected to multiple codebases (for example, a web React library, an iOS SwiftUI library, and an Android Compose library). Each platform lives in its own repository with its own `figma.config.json`. The `label` field is what distinguishes them in Figma's Inspect panel dropdown.

The skill handles this: it confirms which platform to start with, completes the full setup for that one, then reminds the user to repeat the process for each additional platform.

---

## Two starting paths

**Starter kit** - for users starting fresh. The skill points them to a ready-made repository with config files already in place, so they only need to fill in their Figma file key and access token.

**Existing codebase** - for users adding Code Connect to a real project. The skill generates all config files from scratch with real values already populated.

---

## Claude Code integration

If the user has [Claude Code](https://claude.ai/code) installed, the skill switches to agentic mode and Claude Code runs all terminal commands on their behalf. If they don't have it yet, the skill walks them through installing it first. If they prefer to skip it, the skill provides exact terminal commands to copy and run manually, with instructions for both Mac and Windows.

---

## Installing the skill

1. Download `code-connect-setup.skill` from the [Releases](../../releases) page
2. Open the Claude desktop app
3. Drag the `.skill` file into any conversation, or install it via **Settings > Skills**

---

## Using the skill

Once installed, trigger it by asking Claude something like:

- *"Help me set up Code Connect"*
- *"I want developers to see real code snippets in Figma"*
- *"Connect my Figma components to my codebase"*

Claude will open with a short intro and guide you through the rest.

---

## Files generated by the skill

| File | Purpose | Safe to commit |
|------|---------|----------------|
| `figma.config.json` | Tells the CLI which Figma file to connect to and how to parse your code | Yes |
| `tsconfig.json` | Adds Figma type definitions to your editor | Yes |
| `.env` | Stores your Figma access token | No - add to `.gitignore` |
| `*.figma.ts` | One file per connected component | Yes |
